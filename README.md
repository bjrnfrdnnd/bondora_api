## Generation of package
1. generate yaml from swagger spec json (https://api.bondora.com/swagger/docs/v1)
   - use swagger editor on the internet for that
   - save the resulting yaml in /some/path/to/yaml
2. edit yaml to use apiKey instead of oauth2 authentication
   - open the yaml and exchange the securityDefinitions with:
    ```
    securityDefinitions:
      apiKey:
        type: apiKey
        name: Authorization
        in: header
    ```
3. generate python client code:
    ```
    swagger-codegen
    java -jar swagger-codegen/modules/swagger-codegen-cli/target/swagger-codegen-cli.jar generate -i bondora_swagger.yaml -l python -o trading/p2p/bondora/bondora_api
    ```
4. allow api access from bondora's side
   - goto https://api.bondora.com/Login?ReturnUrl=%2fApplication
   - create a new application
   - set "https://www.someapp.com" as "Homepage URL" and as "Authorizaton callback URL"
   - generate a "non-expiring token"
   - use that as apiKey
